**What it does:** SWE-Exp is an experience-enhanced approach for repository-level software issue resolution. It distills concise, reusable "experiences" from prior agent repair trajectories (both successful and failed) and stores them in a multi-faceted Experience Bank, then retrieves and reuses them to guide new repairs. The framework is a four-stage pipeline (Fig. 2): (1) Trajectories Collection from popular open-source repositories, where each repair attempt is represented as tuples ⟨d_t, a_t, s_t+1, f_t⟩ and annotated success/failure; (2) Experiences Extraction by an "Experiencer" agent producing experiences with two components — Perspective (abstract understanding of the problem) and Modification (generalized strategy to fix it), indexed by Issue Type and Description, embedded and stored in a vector database (Experience Bank); experiences are categorized into Comprehension Experiences and Modification Experiences; (3) Experience Retrieval, retrieving top N most relevant experiences by vector similarity then a rerank agent selecting K; (4) Experience Reuse via a dual-agent architecture — an Instructor (high-level strategy/direction: search, view, modify, or finish) and an Assistant (low-level execution) — built on top of the MCTS framework (UCT). Built by extending the SWE-Search framework.

## Evaluation setup
Dataset: SWE-Bench-Verified (500 human-verified issues from SWE-bench). Quote (Sec 4.2): "We evaluate SWE-Exp on the SWE-Bench-Verified dataset [24], which contains 500 verified issues from SWE-bench [16]." "All baseline methods are evaluated on the same dataset." Metric: Pass@1 resolution rate — "We measure the performance based on widely used metric Pass@1 for issue resolution. This metric captures the proportion of issues that are correctly fixed on the first attempt" (Sec 5.1). Models: DeepSeek-V3-0324 and Claude-4-Sonnet (20250514). Quote (Sec 4.4): "We employ both DeepSeek-V3-0324 [7] and Claude-4-Sonnet [1] as our agent models. For DeepSeek-V3-0324, the agent is configured with a temperature of 0.7 and a maximum of 20 iterations, while the remaining configurations follow SWE-Search [2]. We further set the maximum number of finished nodes to 2... For Claude-4-Sonnet, we directly follow the SWE-Search [2] setup." Scaffold: "We implement SWE-Exp by extending the SWE-Search [2] framework with our components." Baselines (Sec 4.3): Agentless, SWE-Agent, SWE-Search, OpenHands, EvoCoder, SAGE. Retrieval (Sec 4.4): "During experience retrieval, we first identify the top N = 10 issues that are most relevant in terms of error type based on cosine similarity. Subsequently, a dedicated reranking agent evaluates these candidates and selects k = 1 most applicable experience... We compute cosine similarity based on embeddings generated by the Multilingual-E5-Large model." Leakage exclusion: "During the experience collection phase, we run SWE-Search on the SWE-Bench-Verified benchmark, collect the resulting trajectories, and extract structured experiences from them. ... To prevent data leakage, we exclude all experiences originating from the same code repository and generated after the target instance." Ablations conducted within DeepSeek-V3-0324 (Table 2). Hyperparameter studies (RQ3): number of experiences 0-4 (Fig 4a); experience bank size studied on a 75-instance subset (25 each from Django, SymPy, Sphinx) with DeepSeek-V3-0324, increments of 100 (Fig 4b). Manual verification of extracted experiences on a subset of 75 instances (Sec 6.2).
