## What a QnA task is (verified by reading instructions + rubrics)

- The repo is mounted at `/app`; the agent explores/runs it and writes a final answer to
  `/logs/agent/answer.txt`. No code edits.
- Questions are design/architecture narratives ("trace X end-to-end", "how does Y work").
- Rubric items are **specific and code-grounded** (exact function chains, byte formats, path
  conventions). A capable no-memory agent can reach many by reading code — so the MCTS-Mem
  effect must come from faster/correct orientation and cross-cutting rationale, measured as
  higher must-have item recall (and, harder, binary pass).

## Picking implication

Highest design-density × best single-build amortization = **kovidgoyal/kitty** (26 tasks, 85%
design). It is the sharpest pilot: if distilled design memory can't lift the agent here, it
won't elsewhere. Scaling toward N≈70 (a publishable headline) favors the high-design,
tractable repos next: paperless-ngx, scapy, simple-login, grafana.
